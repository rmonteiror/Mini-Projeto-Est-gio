Proposta de Integração e Automação Total para Melhorar e Resolver Processos no Atendimento Digital com WhatsApp e Robôs

Feito por: Ramon Monteiro

1. Problema: Redirecionar o WhatsApp Não Oficial para o Oficial

Descrição: Hoje existe um WhatsApp não oficial que está sendo utilizado para comunicação com os clientes, mas a ideia é transferir esses atendimentos automaticamente para o WhatsApp oficial, que utiliza a API oficial do Meta, sem interrupções na experiência do cliente.

Soluções Propostas:

  • Automação com Middleware de Comunicação:
    • Desenvolver um middleware que recebe as mensagens do WhatsApp não oficial e redireciona para o WhatsApp oficial via API.
    • Implementar o uso de APIs como Twilio, Zenvia ou Gupshup, que permitem roteamento e transferência de mensagens entre diferentes plataformas.
  • Integração com HUG (Plataforma Oficial):
    • Criar um webhook no WhatsApp não oficial para capturar mensagens e enviá-las automaticamente ao HUG.
    • Configurar uma camada de integração que conecte o robô do WhatsApp não oficial ao oficial usando a API oficial do Meta.
  • Solução Temporária:
    • Incluir mensagens automáticas no WhatsApp não oficial orientando os clientes a continuarem o atendimento no WhatsApp oficial (linkando o número oficial diretamente com o botão do WhatsApp).

2. Problema: Identificar Clientes Aptos ao Saque FGTS e Criar um Banco de Dados Centralizado

Descrição: O robô atual coleta os leads de clientes potenciais e verifica quem está apto ao saque do FGTS. No entanto, é necessário construir uma base de dados consolidada que mantenha informações atualizadas sobre quais clientes estão aptos mês a mês.

Soluções Propostas:

  • Criação de um Banco de Dados Relacional:
    • Desenvolver um banco de dados centralizado em SQL (MySQL, PostgreSQL) para armazenar informações dos clientes (CPF, status FGTS, saldo, data de liberação).
    • Atualizar a base mensalmente com base nos dados coletados pelo robô.
  • Automação de Consulta e Atualização:
    • Criar um script de integração para o robô consultar automaticamente os CPFs e atualizar o banco de dados com o status FGTS.
    • Implementar notificações automáticas no WhatsApp oficial via API quando um cliente fica apto ao saque.
  • Dashboard de Gestão de Dados:
    • Desenvolver um painel visual (em ferramentas como Power BI, Tableau ou dashboards web customizados) para que os gestores acompanhem em tempo real o status dos clientes aptos ao saque.

3. Problema: Consulta Manual de CPF e Saldo FGTS

Descrição: Atualmente, o robô coleta CPFs de forma específica, mas a consulta do saldo para saque FGTS é feita de forma manual. O objetivo é automatizar essa consulta.

Soluções Propostas:

  • Automação Total da Consulta:
    • Criar um módulo de integração com o robô que utiliza APIs de terceiros ou serviços bancários para verificar automaticamente os saldos disponíveis para FGTS com base nos CPFs.
  • Consulta Manual Simplificada:
    • Caso não seja possível automatizar totalmente, desenvolver uma interface web simples onde os operadores podem inserir o CPF e verificar diretamente o saldo no robô sem acessar outras plataformas.
  • Notificações Proativas:
    • Configurar notificações automáticas para avisar clientes diretamente no WhatsApp oficial sobre a liberação do saldo, reduzindo a necessidade de consulta manual.

4. Problema: Vinculação da URA com API Digital e QR Code

Descrição: Não é possível vincular a URA diretamente à API digital ou ao QR Code, dificultando o direcionamento dos clientes ao WhatsApp oficial.

Soluções Propostas:

  • URAs Inteligentes Integradas com API: Configurar a URA para que as chamadas sejam redirecionadas para um endpoint da API oficial do WhatsApp (Meta), garantindo que a interação se mantenha dentro da estrutura da plataforma oficial.
  • Usar ferramentas como Twilio ou Dialogflow para criar uma URA híbrida que transita entre chamadas e mensagens WhatsApp.
  • QR Code Dinâmico Vinculado ao WhatsApp Oficial: Gerar QR Codes dinâmicos que redirecionam os clientes diretamente para o WhatsApp oficial, eliminando a necessidade de múltiplas integrações complexas.
  • Automação de Fluxo entre URA e HUG: Configurar uma API intermediária que capture os dados do cliente ao interagir com a URA e redirecione automaticamente ao HUG para continuidade do atendimento.

Ferramentas Recomendadas

Descrição: Abaixo estão as principais tecnologias recomendadas para implementar as soluções de automação, integração e monitoramento descritas nos problemas anteriores.

Lista de Ferramentas:

  • APIs e Integração: Twilio, Zenvia, Meta API (WhatsApp Oficial), Dialogflow.
  • Banco de Dados: MySQL, PostgreSQL.
  • Desenvolvimento Web: Node.js, Python (Flask/Django), React para o dashboard.
  • Automação: Zapier, Make (Integromat).
  • Monitoramento: Grafana, Prometheus.
